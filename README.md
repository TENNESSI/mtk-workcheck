# Flask-приложение для учёта техники, рабочих мест и календаря

Готовое веб-приложение на Flask для:

- учёта сотрудников и ролей (`админ` / `сотрудник`)
- учёта отделов сотрудников
- учёта рабочих мест сотрудников
- учёта техники: компьютеры, мониторы, мыши, клавиатуры, наушники и любые другие позиции
- привязки техники к сотруднику и рабочему месту
- контроля даты последнего техосмотра компьютеров
- учёта склада картриджей, включая остаток и количество, ожидающее заправку
- календаря мероприятий с **личным** и **общим** календарём
- назначения **ответственного** за каждое мероприятие
- выбора участников мероприятия с кнопками:
  - выбрать всех
  - снять всех
  - выбрать весь отдел одним кликом
- уведомлений на сайте, если сотрудника добавили в мероприятие или изменили событие

## Что умеют роли

### Администратор
- добавлять, редактировать и удалять сотрудников
- указывать отдел сотрудника
- добавлять и редактировать рабочие места
- добавлять, редактировать и удалять технику
- выдавать технику сотрудникам и привязывать её к рабочему месту
- вести журнал техосмотров
- вести склад картриджей
- видеть все личные и общие события
- создавать, редактировать и удалять любые события календаря

### Сотрудник
- входить в систему
- видеть своё рабочее место и выданную технику
- открывать календарь
- создавать личные события, которые видит только он
- создавать общие события для общего календаря
- выбирать участников общих мероприятий
- редактировать и удалять **только свои** мероприятия
- получать и читать уведомления

## Календарь

В календаре теперь есть:

- `Мой личный` — события видит только владелец календаря
- `Общий` — события видят все сотрудники
- `Все доступные` — объединённый вид

При создании события есть чекбокс **«Добавить в общий календарь»**.

Если он включён:
- можно выбрать ответственного
- можно выбрать участников
- можно выбрать всех сотрудников одной кнопкой
- можно выбрать весь отдел одной кнопкой
- участникам приходят уведомления

Также есть отдельная страница просмотра мероприятия без перехода в режим редактирования.

## Технологии

- Flask
- Flask-Login
- Flask-SQLAlchemy
- SQLite
- Bootstrap 5

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows
pip install -r requirements.txt
python app.py
```

Откройте в браузере:

```text
http://127.0.0.1:5000
```

## Первый вход

По умолчанию создаётся администратор:

- логин: `admin`
- пароль: `admin123`

Можно задать свои данные через переменные окружения перед первым запуском:

```bash
export DEFAULT_ADMIN_USERNAME=superadmin
export DEFAULT_ADMIN_PASSWORD=StrongPassword123
export DEFAULT_ADMIN_NAME="Главный администратор"
```

## Хранилище данных

База создаётся автоматически в:

```text
instance/app.db
```

## Важно при обновлении старой версии

Если у вас уже была старая база `instance/app.db`, приложение попробуется автоматически добавить новые поля для календаря и отдела сотрудников.

Если вы тестировали старую незавершённую версию и хотите начать с чистого листа, просто удалите файл:

```text
instance/app.db
```

после чего запустите приложение заново.
